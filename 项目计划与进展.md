## 一、课题内容

实现一个**快捷键唤起、插件可扩展、数据可同步、跨平台**的桌面提效工具箱，对标 uTools/Alfred/Raycast。

核心解决三大痛点：

1. **入口分散**——统一全局快捷键唤起入口（< 300ms）。
2. **能力固化**——开放插件生态（市场分发 + 本地开发安装）。
3. **数据孤岛**——设置 / 剪贴板 / 插件配置端到端加密跨设备同步。

**三个 P0 内置应用**：时间戳转换、历史剪贴板、截图；**P1 增量**：数据同步（设置/插件）、局域网共享。

> 插件机制 P0 = **本地加载 + 内置列表 UI + 预留 registry 接口（mock JSON）**；真实服务端应用市场（上传/安装/审核/签名）为【挑战】。剪贴板跨设备同步为 P0，设置/插件同步为 P1。

## 二、阶段划分

整体周期 **2 周 / 10 个工作日 / 4 个里程碑 / 单 Sprint**，团队 8 人。原则：先骨架后血肉、先主链路后增量、安全与质量内建、**P0 优先 P1 后置**，每个里程碑产出**可演示、可测试**的增量。

### 第一阶段（D1–D3）：地基 + MVP 主链路

> 对应 **M0 脚手架 + M1 MVP**

- Git 协作模式、仓库与分支规范、开发环境搭建、快速入门
- 完成需求文档（PRD）与技术方案（选型/架构）设计，队长协商分工
- Monorepo + CI + 空壳应用一键启动（CI 全绿）
- 打通主链路：**全局快捷键 → 命令面板 → 时间戳功能（内置 Provider）**（经 `CommandProvider` 统一契约接入，非插件运行时；M2 插件系统复用同一契约）
- 深浅色 / 多语言基础体验
- **退出标准**：FR-001/002/004/005 + 时间戳应用通过验收

### 第二阶段（D4–D10）：可扩展 → 生态 → 收尾加固

> 对应 **M2 Alpha + M3 Beta + M4 RC/1.0**

- **M2 Alpha（D4–D5）可扩展内核**：插件系统（加载/隔离）+ 内置列表 + mock registry + 本地开发安装 + 悬浮球 + 换肤
  - 退出标准：FR-003/006/010/012 + 插件加载/隔离通过
- **M3 Beta（D6–D8）P0 应用收尾**：剪贴板（含 E2EE 同步）+ 截图（选区/贴图）
  - 退出标准：FR-040~043/060/061 通过（三个 P0 应用齐备）
- **M4 RC/1.0（D9–D10）收尾加固 + 发布**：安全基线 + 自动更新 + 全量回归发布
  - 退出标准：安全基线 + 全量回归通过
  - **挑战/P1 后置/选做**（仅 P0 全绿且行有余力时做，不阻塞发布）：应用市场服务端（上传/安装/审核）、市场更新、截图标注/马赛克、数据同步（设置/插件 P1）、局域网（P1）→ FR-011/014/015/062/063/030/031/050/051
- 队长在 **D5 中期校准**（Go/No-Go 是否降级 P1）与 **D10 Sprint Review / Retro** 汇总进度与风险
